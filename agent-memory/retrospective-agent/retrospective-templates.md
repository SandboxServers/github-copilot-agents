## Retrospective Templates

### Sprint Retrospective Template (30 minutes, team level)

**Pre-Work Checklist for Facilitator:**
- [ ] Gather sprint metrics: velocity, burndown, deployment count, defect count
- [ ] Review previous sprint's action items and their current status
- [ ] Prepare shared document with agenda and previous action items pre-populated
- [ ] Send reminder to team with pre-read materials 24 hours before

**Agenda:**
1. **Previous action items** (5 min) — review status of each, celebrate completions
2. **What went well** (8 min) — each team member shares one item, brief discussion
3. **What didn't go well** (8 min) — each team member shares one item, no blame
4. **What to change** (7 min) — propose and vote on 1-3 specific changes for next sprint
5. **Capture action items** (2 min) — assign owner and due date to each

**Facilitation Notes:**
- Use round-robin to ensure everyone speaks
- Timebox each section strictly — better to be brief than to overrun
- Limit to 1-3 action items — fewer items completed is better than many items ignored
- Record the retro notes in the team's standard location immediately

**Output Format:**
- 1-3 action items with owner, due date, and success criteria
- Summary of key discussion points (3-5 bullet points)
- Updated status on previous action items

### Engagement Retrospective Template (90 minutes, cross-team)

**Pre-Work Checklist for Facilitator:**
- [ ] Compile planned vs actual analysis (timeline, cost, scope, quality)
- [ ] Gather metrics: cost actuals, defect counts, deployment frequency, SLA adherence
- [ ] Collect security review findings summary and resolution status
- [ ] Review all sprint retrospective action items from the engagement
- [ ] Identify stakeholders to invite from each workstream
- [ ] Prepare timeline of key events, decisions, and milestones
- [ ] Send pre-read package to participants 48 hours before

**Agenda:**
1. **Context setting** (5 min) — engagement scope, goals, and timeline summary
2. **Previous engagement action item review** (10 min) — status and outcomes
3. **Planned vs actual walkthrough** (15 min) — data-driven comparison
4. **What went well** (15 min) — successes across all workstreams
5. **What didn't go well** (15 min) — gaps, surprises, and failures
6. **Root cause analysis** (15 min) — Five Whys or Fishbone for top 2-3 issues
7. **Patterns and systemic observations** (10 min) — cross-engagement trends
8. **Action items and next steps** (5 min) — assign owners and due dates

**Facilitation Notes:**
- Multiple workstreams means multiple perspectives — allocate time for each
- Use the planned vs actual data as the anchor, not individual recollections
- Focus root cause analysis on the 2-3 highest-impact issues, not every issue
- Ensure action items span process, tooling, training, and architecture categories

**Output Format:**
- Planned vs actual comparison table
- Root cause analysis for top issues
- 5-8 action items with owners, due dates, and success criteria
- Patterns to add to organizational knowledge base
- Recommendations for future engagements

### Incident Post-Mortem Template (60 minutes, blameless)

**Pre-Work Checklist for Facilitator:**
- [ ] Construct incident timeline from monitoring data, chat logs, and ticket history
- [ ] Identify all responders and stakeholders to include
- [ ] Gather detection metrics: time to detect, time to respond, time to resolve
- [ ] Collect customer impact data if applicable
- [ ] Confirm blameless ground rules will be stated at the start
- [ ] Schedule within 72 hours of incident closure while details are fresh

**Agenda:**
1. **Ground rules and context** (5 min) — state blameless principles, summarize incident
2. **Timeline walkthrough** (15 min) — fact-based sequence of events
3. **Detection and response** (10 min) — how was it detected? What was the response path?
4. **Root cause analysis** (15 min) — Five Whys from the symptom to the systemic cause
5. **What would have helped** (10 min) — better monitoring, runbooks, architecture changes
6. **Action items** (5 min) — prevention, detection, and response improvements

**Facilitation Notes:**
- Start with explicit statement: "We are here to learn, not to blame"
- Stick to the timeline — facts and data, not memory and opinion
- Ask "what did you know at the time?" not "why didn't you do X?"
- Ensure action items cover all three areas: prevent, detect, respond faster

**Output Format:**
- Incident summary (what happened, impact, duration)
- Validated timeline of events
- Root cause analysis with Five Whys chain
- 3-5 action items categorized as prevention, detection, or response
- Updates needed for runbooks, monitoring, or architecture
