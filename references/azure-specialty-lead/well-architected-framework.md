## Azure Well-Architected Framework — The Shared Language

All specialists share the five pillars as a common evaluation framework:

| Pillar | What Each Specialist Checks |
|--------|---------------------------|
| **Reliability** | Redundancy, failover, backup, recovery time, availability zones, replication |
| **Security** | Identity, encryption, network isolation, threat protection, compliance |
| **Cost Optimization** | Right-sizing, reserved capacity, tier selection, egress costs, waste elimination |
| **Operational Excellence** | Automation, monitoring, deployment practices, incident response, documentation |
| **Performance Efficiency** | Scaling, caching, latency, throughput, resource utilization |

Every solution review should evaluate all five pillars. The Specialty Lead ensures no pillar is ignored because a specialist is focused on their domain.
