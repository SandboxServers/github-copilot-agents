## Organizational Challenges

### Shadow IT
- **What it looks like**: Teams created without governance, Power Apps built without admin awareness, third-party file sharing instead of OneDrive, personal email forwarding, ungoverned AI tools
- **Why it happens**: Official tools are too locked down, too slow to provision, or users don't know they exist
- **How to address it**: Make the right thing easy. Self-service with guardrails, not prohibition. Cloud App Security (Defender for Cloud Apps) to discover shadow IT. Power Platform DLP policies to govern citizen development.

### User Adoption
The most secure, well-governed platform no one uses is a failure. Adoption requires:
1. **Executive sponsorship**: Visible leadership using the platform
2. **Scenarios, not features**: Train on "how to run a project" not "how to use Planner"
3. **Champions network**: Peer advocates in each department
4. **Measured adoption**: Track usage via M365 usage reports, Viva Insights
5. **Feedback loops**: Regular surveys, champion roundtables, quick wins to build momentum
6. **Training**: Microsoft 365 learning pathways (customizable, on-demand), role-based training paths

### Change Management
- **Communication plan**: What's changing, why, when, who's affected, where to get help
- **Phased rollout**: Pilot group → early adopters → broad deployment
- **Support readiness**: Help desk trained before users encounter changes
- **Success metrics**: Define before rollout — what does good look like?
