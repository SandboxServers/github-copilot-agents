## Error Handling — The Complete Picture

### Three Error Types
1. **Non-terminating**: pipeline continues, caught by `catch` only when escalated. Generated by `Write-Error`, `$PSCmdlet.WriteError()`
2. **Statement-terminating**: stops current statement, next statement runs. Generated by `.ThrowTerminatingError()`, engine errors, .NET exceptions
3. **Script-terminating**: unwinds call stack. Generated by `throw`, parse errors, escalated non-terminating errors

### The ErrorAction / $ErrorActionPreference Asymmetry
```
-ErrorAction parameter → overrides $ErrorActionPreference for ONE command (non-terminating only)
$ErrorActionPreference → applies to ALL commands in scope AND child scopes
```
- `-ErrorAction Stop` on non-advanced function → escalates to **script-terminating**
- `-ErrorAction Stop` on advanced function (`[CmdletBinding()]`) → escalates to **statement-terminating**
- `$ErrorActionPreference = 'Stop'` in non-advanced context → script-terminating
- `$ErrorActionPreference = 'Stop'` in advanced function → statement-terminating
- `.ThrowTerminatingError()` ignores `-ErrorAction` (except `Break`)
- `throw` can be suppressed by `$ErrorActionPreference = 'SilentlyContinue'` — surprising but true

### Production Error Handling Pattern
```powershell
function Invoke-SafeOperation {
    [CmdletBinding()]
    param(
        [Parameter(Mandatory)]
        [string]$ResourceName
    )
    try {
        $result = Get-AzResource -Name $ResourceName -ErrorAction Stop
    }
    catch [Microsoft.PowerShell.Commands.HttpResponseException] {
        Write-Warning "HTTP error for $ResourceName`: $($_.Exception.Message)"
        return $null
    }
    catch [System.Management.Automation.ItemNotFoundException] {
        Write-Warning "Resource not found: $ResourceName"
        return $null
    }
    catch {
        Write-Error "Unexpected error processing $ResourceName`: $_"
        throw  # Re-throw to caller
    }
}
```

### Native Command Error Handling (PS 7.4+)
```powershell
$PSNativeCommandUseErrorActionPreference = $true
$ErrorActionPreference = 'Stop'
try {
    git clone https://example.com/repo.git
} catch {
    Write-Error "Git failed with exit code $LASTEXITCODE"
}
```

### Error State Variables
- `$?` — `$true` if last operation succeeded, `$false` otherwise
- `$Error` — `ArrayList` of recent error records (most recent at `[0]`), max `$MaximumErrorCount` (256)
- `$LASTEXITCODE` — exit code of last native program (`0` = success)
- `$Error.Clear()` — reset error collection (useful in testing)
