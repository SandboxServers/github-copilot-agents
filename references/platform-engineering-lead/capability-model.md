## Platform Engineering Capability Model

Microsoft's model defines six core capabilities. Use the model to assess where the organization is today and set goals for improvement.

### The Six Capabilities
| Capability | Key Question |
|---|---|
| **Investment** | How are staff and funds allocated to platform capabilities? |
| **Adoption** | Why and how do users discover and use the platform? |
| **Governance** | How do you ensure users have access to needed resources while managing costs, data, and IP? |
| **Provisioning & Management** | How do users create, deploy, and manage resources? |
| **Interfaces** | How do users interact with and consume platform capabilities? |
| **Measurement & Feedback** | What's the process for gathering feedback and measuring success? |

### Maturity Stages Per Capability
| Stage | Description |
|---|---|
| **Provisional** | Ad hoc, informal, individual efforts |
| **Defined** | Standard tooling, documented processes, still requires domain expertise |
| **Managed** | Self-service solutions, consistent interfaces, minimal support needed |
| **Optimized** | Automated, data-driven, continuously improving |

### Assessment Approach
- You don't need to advance all capabilities simultaneously
- Chart a course that makes sense for the organization
- Most organizations are more advanced in some capabilities than others
- Focus investment where it delivers the most value to internal customers
