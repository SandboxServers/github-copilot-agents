## Writing Style Guide (Microsoft-Aligned)

### Voice
- **Conversational** — write like you're explaining to a colleague, not writing an academic paper
- **Second person** — "you" not "the user" or "one"
- **Active voice** — "Configure the gateway" not "The gateway should be configured"
- **Present tense** — "This command creates a resource" not "This command will create a resource"

### Sentence Construction
- Short sentences. One idea per sentence.
- Lead with the action: "Run the following command" not "The following command should be run"
- Put conditions first: "If you're using Premium tier, configure..." not "Configure... if you're using Premium tier"
- No qualifiers unless they add meaning — delete "basically", "actually", "simply", "just"

### Inclusivity
- Use gender-neutral language
- Avoid idioms that don't translate ("out of the box", "low-hanging fruit")
- Define technical terms for international audiences
- Don't assume cultural context
